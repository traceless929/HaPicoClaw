# Home Assistant Tools

本工作区默认通过 Home Assistant 官方 `mcp_server` 提供家庭控制能力，并由 PicoClaw 将其注册为带 `mcp_homeassistant_` 前缀的实际工具名。

重要说明：

- 不要假设后缀工具名固定不变，优先使用当前会话里实际可见的 `mcp_homeassistant_*` 工具。
- 先看工具描述，再决定用哪个；不同 Home Assistant 版本或暴露配置下，可用工具集合可能不同。
- 对控制类请求，优先先查询状态，再执行写操作。

## 使用顺序建议

1. 先判断用户是在问状态、找设备、执行控制，还是排查历史。
2. 不确定目标实体时，优先选择能列出实体、设备或场景的 `mcp_homeassistant_*` 工具。
3. 要执行控制时，优先选择读取当前状态的 `mcp_homeassistant_*` 工具。
4. 只有目标明确且工具描述表明可以执行控制时，才调用写操作工具。
5. 需要回溯最近发生过什么时，优先选择历史、日志或最近变化相关工具。

## 常见注意事项

- Home Assistant 官方 MCP 的工具集合来自 HA 当前暴露的能力与实体配置。
- 如果用户只是问“开了吗”，不要直接执行写操作。
- 如果工具报错，优先把错误原因翻译成用户能理解的说明，再给排查建议。
- 如果用户提到的设备不明确，先列候选项或请用户澄清，不要猜测具体实体。
